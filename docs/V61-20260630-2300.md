# V61 — 去美团化 & 数据库迁移

> 日期：2026-06-30
> 范围：全项目代码清理 + 数据库切换 + CORS 修复

---

## 变更概览

本次变更将 WorkBuddy 从美团内部 NoCode 平台剥离为独立项目，去除所有内部依赖，切换到外部 NoCode 数据库。

---

## 1. 数据库迁移

**旧数据库**：`https://db0e6ev0fkkmoc.database.sankuai.com`
**新数据库**：`https://dbc23lmh865kibbhuu.database.nocode.cn`

在新数据库上创建了全部 9 张表：tasks、memos、task_members、task_tags、task_groups、task_comments、task_notes、reading_items、quick_notes。

开发环境通过 Vite proxy 代理数据库请求（`/supabase-api` → 远程数据库），解决浏览器 CORS 限制；生产环境直连远程数据库。

**修改文件**：
- `src/integrations/supabase/client.js` — 更新 URL/Key，开发环境走 proxy
- `vite.config.js` — 新增 `/supabase-api` proxy 配置

## 5. 开发调试说明

```bash
# 安装依赖
yarn

# 启动开发服务器（默认 8080 端口）
yarn dev

# 访问
open http://localhost:8080
```

开发环境下数据库请求通过 Vite proxy 代理，无需额外配置。如遇 CORS 错误，确认 `vite.config.js` 中 proxy 配置存在且 dev server 已重启。
